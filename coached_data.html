Overview

This artifact focuses on a practical machine learning question that often gets treated too casually: how to decide whether an imputation method is acceptable before using it in a final predictive model. Instead of choosing a fill strategy by convenience, I designed a sensitivity-analysis framework to test whether the imputation would meaningfully alter model behaviour.

The project was built through an interactive AI-coaching workflow. The chatbot did not simply produce an answer; it helped me refine the structure of the experiment, challenge unclear reasoning, and translate the method into working Python code. The result was a reproducible evaluation process with interpretable metrics and a defensible imputation decision.

Why this artifact matters

“In analytics and machine learning, small preprocessing decisions can have meaningful downstream effects, and those decisions deserve evidence, not convenience.”

Challenge and Dataset

Problem

If a feature contains missing values, how can we test whether a chosen imputation method changes predictions too much to trust?

Dataset Choice

I selected the UCI Student Performance (Portugal) dataset and worked with the student-por.csv file.

Modeling Setup

The target was G3 (final grade), and the imputation column was studytime, with missingness introduced for the test framework.

Method and Experimental Design

The core of the project was a sensitivity-analysis workflow. I did not want to assume that mean imputation was acceptable simply because it was easy to apply.

  1. Create the testing structure Treat the full dataset as D0. Build a complete-case subset D1, then split it into S_base and S_mask.
  2. Train a baseline model Use S_base with true values to train the model and generate baseline predictions R0 for the masked subset.
  3. Apply imputation and retrain Impute the masked values in S_mask using mean imputation, combine S_base + imputed S_mask, retrain the model, and generate new predictions R1.
  4. Measure prediction shift Compute the distribution of Δ = R1 - R0 and summarize it using mean shift, standard deviation, mean absolute shift, and the 95th percentile.
  5. Judge acceptability Compare the statistics against thresholds and use RMSE to determine whether the imputation method is stable enough.

Why Option 1 was chosen

I selected the version that retrained the model after imputation because I wanted to test how imputed values affect model learning itself, not only how they affect a single prediction pass.

Role of the AI coach

The chatbot was most useful in clarifying the structure of the experiment, prompting me to choose a primary metric, and helping convert the final design into executable Python code. I also corrected the chatbot when it misunderstood my initial procedure, which made the collaboration more iterative and technically meaningful.

Implementation

The experiment was implemented in Python using a RandomForestRegressor, simple mean imputation, cross-validation, and RMSE-based evaluation. The technical process also involved debugging environment issues, including an error related to mean_squared_error and a scoring-string issue in scikit-learn. That debugging work was part of the real learning value of the exercise.

Threshold outcome

All threshold checks passed. Under this framework, the tested mean-imputation method was considered acceptable for the selected feature and dataset.

Decision

Based on the evidence, the method did not introduce large or unstable prediction shifts, so it could be justified for final use with documented assumptions.

What I Learned

This challenge changed the way I think about missing data in machine learning. It showed me that even a simple imputation method can be examined more carefully through a structured experiment instead of being accepted as a routine preprocessing step.

It also reinforced that AI support is most valuable when used critically. The strongest part of the process was not instant answer generation, but the iterative refinement of definitions, assumptions, thresholds, and implementation details.
