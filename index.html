Artifact 4

AI-Coached Data Challenge

Sensitivity analysis for missing-value imputation

Project Page

Introduction

An AI-guided ML challenge built around a practical data-quality problem: deciding whether an imputation method is acceptable before using it in a final model.

Description

Using the UCI Student Performance dataset, I designed a sensitivity-analysis workflow to test whether mean imputation for the studytime variable would introduce unacceptable variation in predicted student grades.

Objective

To develop a more defensible approach to missing-value handling by measuring how imputation affects model learning and prediction stability.

Process

  • Defined a testing framework using D0, D1, S_base, and S_mask
  • Worked with an AI coaching bot to refine the experiment
  • Implemented the workflow in Python
  • Evaluated using RMSE, delta statistics, and cross-validation

Tools & Tech

Python, pandas, NumPy, scikit-learn, RandomForestRegressor, SimpleImputer, cross-validation, SchoolAI coaching support.

Unique Value

Validates the effect of imputation itself rather than assuming the method is harmless — a more reproducible and defensible approach.

References

  • Cortez, P., & Silva, A. M. G. (2008). Using data mining to predict secondary school student performance.
  • UCI Student Performance dataset documentation. Artifact project page and implementation summary.
Artifact 5

AutoWise Finder

A decision-support tool for used car selection

Project Page

Introduction

An interactive decision-support prototype to help used-car buyers compare options more thoughtfully than a simple price or mileage sort.

Description

Asks for buyer preferences — budget, monthly payment, seating, mileage, accident tolerance, reliability, resale value, and comfort — then ranks vehicles via a transparent weighted scoring model.

Objective

To design a practical AI/ML-inspired recommendation artifact supporting better consumer decision-making through scoring, ranking, cost estimation, and explainable recommendations.

Process

  • Defined buyer decision problem and key trade-off factors
  • Created a weighted scoring model for vehicle comparison
  • Built a static GitHub Pages web app using HTML, CSS, and JavaScript
  • Added explainable ranking, payment, resale, and ownership-cost estimates

Tools & Tech

HTML, CSS, JavaScript, GitHub Pages, scoring logic, user-input forms, explainable recommendation design, static web deployment.

Unique Value

Focuses on decision quality — considering affordability, reliability, resale value, family fit, mileage risk, comfort, and ownership cost together.

References

  • AutoWise Finder project page and implementation files.
  • Course concepts: data-driven decision-making, responsible AI design, explainable recommendations.
