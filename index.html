Artifact 4

AI-Coached Data Challenge

Sensitivity analysis for missing-value imputation
Project Page

Introduction

This artifact documents an AI-guided machine learning challenge built around a practical data-quality problem: deciding whether an imputation method is acceptable before using it in a final model.

Description

Using the UCI Student Performance dataset, I designed a sensitivity-analysis workflow to test whether mean imputation for the studytime variable would introduce unacceptable variation in predicted student grades.

Objective

To develop a more defensible approach to missing-value handling by measuring how an imputation choice affects model learning and prediction stability.

Process

  • Defined a testing framework using D0, D1, S_base, and S_mask
  • Worked with an AI coaching bot to refine the experiment
  • Implemented the workflow in Python
  • Evaluated the method using RMSE, delta statistics, and cross-validation

Tools and Tech Used

Python, pandas, NumPy, scikit-learn, RandomForestRegressor, SimpleImputer, cross-validation, SchoolAI coaching support.

Value Proposition of the Artifact

This project shows that I can combine machine learning reasoning, experimental design, and AI-supported problem solving in a disciplined way.

Unique Value

Unlike a standard preprocessing exercise, this artifact focuses on validating the effect of imputation itself rather than assuming the method is harmless.

Relevance

Relevant to employers and reviewers interested in careful model preparation, reproducible evaluation, and thoughtful human-AI collaboration.

References

  • Cortez, P., & Silva, A. M. G. (2008). Using data mining to predict secondary school student performance.
  • UCI Student Performance dataset documentation.
  • Artifact project page and implementation summary.
Artifact 5

AutoWise Finder

A decision-support tool for used car selection
Project Page

Introduction

AutoWise Finder was created as an interactive decision-support prototype to help used-car buyers compare options more thoughtfully than a simple price or mileage sort.

Description

The tool asks for buyer preferences such as budget, monthly payment target, seating needs, mileage limit, accident tolerance, reliability priority, resale value, and comfort preference. It then ranks vehicles using a transparent weighted scoring model.

Objective

To design a practical AI/ML-inspired recommendation artifact that supports better consumer decision-making through scoring, ranking, cost estimation, and explainable recommendations.

Process

  • Defined the buyer decision problem and key trade-off factors
  • Created a weighted scoring model for vehicle comparison
  • Built a static GitHub Pages web app using HTML, CSS, and JavaScript
  • Added explainable ranking, estimated payment, resale estimate, ownership-cost estimate, and real-market search links

Tools and Tech Used

HTML, CSS, JavaScript, GitHub Pages, scoring logic, user-input forms, explainable recommendation design, and static web deployment.

Value Proposition of the Artifact

This artifact demonstrates how AI/ML thinking can be applied to an everyday financial and family decision by converting preferences and constraints into a transparent ranking system.

Unique Value

Unlike a normal car search page, AutoWise Finder focuses on decision quality. It considers affordability, reliability, resale value, family fit, mileage risk, comfort, accident risk, and ownership cost together.

Relevance

Relevant to employers and reviewers interested in applied AI/ML, human-centered decision support, explainable scoring systems, and practical web-based prototypes.

References

  • AutoWise Finder project page and implementation files.
  • Course concepts related to data-driven decision-making, responsible AI design, and explainable recommendations.
